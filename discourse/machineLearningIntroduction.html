02   End-to-End Machine Learning Projects

Keywords and Notations

A sequence of data processing components is called a data pipeline.

A single machine learning model may be part of a much bigger project, with the output of this model being fed into another model as one of its (potentially many) input(s).

Usually the models are somewhat self-contained, some input data comes into a model, and it outputs its result into a data store. This is then pulled into another model which then provides its own ouput. The system can therefore be asynchronous.



A regression task whereby we are trying to predict multiple variables is called a multi-variate regression problem. As such, you can figure out what a univariate regression task is.

Psst..."uni-" means one


When the task involves using multiple input features to predict the output(s). For example, using district population, number of rooms, median income etc. to predict the price of a house.


Root Mean Square Error (RMSE)


Here "m" is the number of instances. "h" is your prediction function (i.e. your model). This equation finds the difference between the actual and predicted value for the i-th instance, squares it, then sums over all instances in your training data.

RMSE gives a higher weight for larger errors.


Let's say given a few features such as height, weight, frequency of exercise we would like to predict the person's life expectancy.

Each person would be an instance.

When we collect the data to train our model, we can represent a person's attributes (height, weight, etc.) with an input vector (x(i)), $$ \bf{x}^{(1)} = \begin{bmatrix} 1.87 \\ 78 \\ 8 \\ 0 \end{bmatrix} $$

I.e. the first person's attributes are: height is 1.87 m, weight is 78 kg, exercises 8 times a month, has never smoked (binary - 0 or 1 meaning yes or no).

For each input vector, there would be a corresponding label (y(i)) - the desired output (in this case it is a scalar value for the age at which the person died.) $$ y^{(1)} = 72 $$


We can use a matrix X which hold all the feature vectors of all the instances.
$$ \mathbf{X} = \begin{bmatrix} {(\mathbf{x}^{(1)})}^T \\ {(\mathbf{x}^{(2)})}^T \\ {(\mathbf{x}^{(3)})}^T \\ ... \\ {(\mathbf{x}^{(i)})}^T \end{bmatrix} $$ The vectors are transposed, such that the matrix columns represent a single feature for all the instances.



Mean Absolute Error (MAE) - here instead of the squaring function (as with RMSE) we simply take the absolute of the difference. $$ MAE(\mathbf{X},h) = \frac{1}{m} \sum^{m}_{i=1} |h(\mathbf{x}^{(i)})-\mathbf{y}^{(i)}| $$

They both measure the distance between two vectors: RMSE (Euclidean Norm or L2 norm) and MAE (Manhattan Norm or L1 norm). The value after the L- in L1 or L2 norm represents the power to which each element is taken to, then rooted.

So for L2 we square all elements and square root the average of the squared elements. For L3 we cube all the elements and cube root the average of the cubed elements. For L-k we take it to power k, and k-root the average.

The higher the norm index the more it focuses on large values and neglect smaller ones.

The problem with random sampling to obtain a test set is that the test set may not be representative of the full set, since the data is randomly sampled.

A better solution might be to do statified sampling based on an important feature. Let's say the median income has a big impact on housing prices. Then you would want to make sure to categorise the incomes, and ensure the test and train data have proportional data distribution accross the income brackets. This ensures the test data is representative of the train data, and that the importance of a particular income bracket is not exaggerated. Therefore the evaluation would be more representative of the model.
